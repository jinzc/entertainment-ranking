# 🎬 综合影视榜单聚合

每小时自动采集 **微博 · 豆瓣 · 抖音 · 百度** 四大平台的影视榜单数据。

## 📊 九个榜单

| 排序 | 平台 | 榜单 | 说明 |
|------|------|------|------|
| 1 | **微博** | 📰 文娱热搜榜 | 微博热搜中文娱相关内容 |
| 2 | **微博** | 📺 剧集热度榜 | 微博热门电视剧排行 |
| 3 | **微博** | 🎬 电影热度榜 | 微博热门电影排行 |
| 4 | **豆瓣** | 🎬 实时热门·电影榜 | 豆瓣电影热度排行 |
| 5 | **豆瓣** | 📺 实时热门·电视榜 | 豆瓣电视剧热度排行 |
| 6 | **抖音** | 🎬 影视榜·院线电影 | 抖音电影热度 |
| 7 | **抖音** | 📺 影视榜·剧集 | 抖音电视剧热度 |
| 8 | **百度** | 🎬 热搜·电影榜 | 百度搜索电影热度 |
| 9 | **百度** | 📺 热搜·电视剧榜 | 百度搜索电视剧热度 |

## ✨ 功能特点

- **四大平台聚合**：微博、豆瓣、抖音、百度数据一站式查看
- **平台Tab切换**：点击平台按钮切换不同来源
- **子榜单切换**：每个平台下多个子榜单切换
- **微博默认优先**：打开页面默认显示微博文娱热搜
- **TOP3高亮**：前三名有特殊标识
- **每小时自动更新**：GitHub Actions 定时采集
- **响应式设计**：手机/电脑都能正常浏览

## 🚀 快速部署

### 1. 创建 GitHub 仓库

1. 在 GitHub 创建新仓库（如 `movie-rankings`）
2. 上传所有文件

### 2. 配置 GitHub Pages

1. 进入 **Settings → Pages**
2. Source 选择 **GitHub Actions**

### 3. 启动工作流

1. 进入 **Actions** 标签页
2. 找到 "Update All Rankings" 工作流
3. 点击 **Run workflow** 手动运行一次

### 4. 访问页面

`https://你的用户名.github.io/仓库名/`

## 📁 文件结构

```

├── update_rankings.py              # 爬虫脚本

└── README.md                       # 说明文档
```

## 🔧 技术栈

- **后端**：Python + Requests
- **前端**：原生 HTML/CSS/JavaScript
- **部署**：GitHub Actions + GitHub Pages
- **数据**：JSON 文件存储

## ⚠️ 注意事项

1. 微博数据通过第三方免费API获取，可能存在延迟
2. 豆瓣API可能有访问限制，备用数据会自动填充
3. 抖音榜单目前使用备用数据（可通过抓包获取真实API）
4. 百度数据通过 60s API 获取
5. 建议配合 Gitee 做国内镜像访问
